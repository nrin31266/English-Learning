Tổng quan Hệ thống Học Từ Vựng AI

Tài liệu mô tả chi tiết toàn bộ vòng đời của dữ liệu: Từ thiết kế Database, tự động hóa bằng AI (Python Worker + Kafka), can thiệp của Admin (Human-in-the-loop), đến trải nghiệm của Người học (React Frontend).

1. Tổng quan Mục tiêu

Hệ sinh thái Từ vựng Toàn diện

Hệ thống kết hợp 2 tính năng chính vào một kiến trúc duy nhất:

  • Tra từ điển tự do (Dictionary Lookup): Người dùng bôi đen từ lạ trong bài Reading/Shadowing. Hệ thống tra AI tức thì và lưu vào kho dùng chung.
  • Học theo Chủ đề (Vocab Course): Admin chỉ cần nhập Tên Chủ Đề (Ví dụ: TOEIC Part 1, Web Development). AI sẽ tự động phân rã thành các bài học nhỏ (Subtopic) và sinh danh sách từ vựng chuyên ngành.

2. Thiết kế Database (Entities)

Kiến trúc tách bạch rõ ràng giữa Dữ liệu tĩnh (Dictionary) và Dữ liệu động (Contextual Course), sử dụng MongoDB.

CollectionVai trò cốt lõiCác trường quan trọng
words Kho từ điển gốc (Dùng chung toàn hệ thống). Lưu mọi nghĩa có thể có của một từ. id (key|pos), text, phonetics, definitions[]
vocab_topics Chủ đề lớn (Chứa nhiều Subtopic). title, cefrRange, tags[], isActive
vocab_subtopics Bài học nhỏ (Ranh giới ngữ cảnh hẹp). title, description (AI Gen), isActive
vocab_word_entries Bộ nhớ đệm Ngữ cảnh. Link từ Subtopic đến Word. Fix cứng nghĩa của từ cho đúng bài học đó, giúp Load giao diện O(1). subtopicId, wordKey, pos, contextDefinition, contextExample...

3. Lõi AI & Chiến thuật Prompting

Thiết kế Ranh giới Độc lập (Mutually Exclusive)

Thay vì truyền hàng ngàn từ vào biến avoid để cấm AI sinh trùng lặp (gây tốn Token và AI bị ảo giác), hệ thống dùng chiến thuật "INCLUDES / EXCLUDES":

  • Khi sinh Subtopic, ép AI viết 3-4 câu description mô tả cực rõ ranh giới.
  • Ví dụ: "Chỉ tập trung vào HTML/CSS. Strictly Excludes Figma, Backend, Database."
  • Kết quả: Các Subtopic không dẫm chân lên nhau, từ vựng sinh ra cực kỳ ngách và chính xác.

4. Luồng Vận hành: Tự động hóa (Auto Gen)

Bước 1: Tạo Topic & Phân rã Subtopic (Java -> AI)
Admin tạo 1 Topic. Java gọi AI để đẻ ra N Subtopics. Mỗi Subtopic có một đoạn description làm ranh giới ngữ cảnh tuyệt đối.
Bước 2: Sinh danh sách từ vựng (Async Kafka)
Admin bấm "Gen Words". Java loop qua các Subtopics. AI dựa vào `description` để đẻ ra 20 từ (Over-fetching để bù trừ). Java lưu các từ này dưới dạng `VocabWordEntry` (Status = PENDING).
Bước 3: Công nhân lấp đầy dữ liệu (Python Worker)
Python Worker chạy ngầm, bốc các từ PENDING đem lên AI tra từ điển chuẩn (Phát âm, Nghĩa, Ví dụ). Khi tra xong, Worker bắn Kafka về Java để update cache context vào VocabWordEntry. UI hiển thị số đếm % tiến độ.

5. Luồng Vận hành: Ghi đè Ngữ cảnh (Human-in-the-loop)

Tính năng cứu cánh khi AI lấy sai nghĩa (Ví dụ: Bài học Web Dev nhưng từ "layout" lại dịch thành "Sơ đồ kiến trúc phòng ốc"). Quá trình này diễn ra Đồng bộ (Sync).

Ghi đè thủ công (Overwrite)
Trên UI Frontend, Admin bấm "Sửa Ngữ Cảnh". Hệ thống liệt kê toàn bộ các nghĩa có sẵn của bảng Word. Admin chọn 1 nghĩa đúng -> Update vào cache VocabWordEntry.
Sinh nghĩa mới bằng AI (Generate Sync Meaning)
Nếu kho không có nghĩa đúng, Admin bấm "Tạo nghĩa bằng AI". Backend gọi API Sync sang Python. Python dùng SINGLE_MEANING_PROMPT soi kỹ subtopicDescription để đẻ ra ĐÚNG 1 nghĩa chuyên ngành -> Lưu bổ sung vào kho Word và update Cache.

6. Luồng Kiểm duyệt & Xuất bản (Publishing)

Hệ thống AI không tự ý đưa bài học ra cho sinh viên. Nó tuân thủ quy trình kiểm duyệt:

  • Dù AI đã chạy xong 100% (Trạng thái READY), cờ hiển thị isActive mặc định vẫn là false.
  • Admin vào rà soát nội dung. Dùng luồng Human-in-the-loop để nắn lại các từ chệch ngữ cảnh.
  • Hoàn tất, Admin bật Toggle isActive = true. Lúc này API bên Learner mới fetch được Data.

7. Trải nghiệm Học tập (Study Modes)

Đa dạng phương thức ôn luyện

Do cấu trúc Definition chứa đủ Text, Ví dụ tiếng Anh, Bản dịch tiếng Việt, hệ thống Frontend có thể render ra nhiều Game học tập:

  • Flashcard: Lật thẻ Nhìn tiếng Anh -> Nhớ tiếng Việt.
  • Fill-in-the-blank: Hệ thống đục lỗ từ vựng trong câu `contextExample`, yêu cầu học viên gõ lại.
  • Dictation: Nghe phát âm `phonetics` và gõ lại từ.
  • Quiz Choice: Chọn đáp án dịch nghĩa đúng.

8. Theo dõi Tiến độ (Progress Tracking)

Lưu ở collection user_vocab_progress (có thể nằm ở User Service hoặc Dictionary Service).

  • Tracking cấp độ Subtopic: Lưu mảng learnedWordKeys (từ đã học), reviewWordKeys (từ sai nhiều cần ôn lại).
  • Thuộc tính completionRate (0% -> 100%).
  • Tracking thời gian: lastStudiedAt để tính chuỗi Streak và gửi Notification nhắc nhở.
